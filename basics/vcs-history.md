# A Brief History of Version Control

Version control evolved from local file tracking to distributed systems, driven by the need for better collaboration and reliability.

## Timeline

- `1982 — RCS` — Local VCS. Tracked single files on one machine. No collaboration.
- `1990 — CVS` — Centralized. Built on RCS, added multi-file support and a central server.
- `2000 — SVN` — Centralized. Fixed CVS bugs, became the industry standard for years.
- `2000 — BitKeeper` — Distributed. First popular DVCS. Used by the Linux kernel team, but proprietary.
- `2005 — Git` — Distributed. Created by Linus Torvalds after Linux lost access to BitKeeper. Fast, free, open source.
- `2008 — GitHub` — Hosting platform for Git repos. Added pull requests, issues, and social collaboration. Made Git the industry standard.

## The Key Story

> Linus Torvalds built Git in ~10 days after the Linux kernel team lost free access to BitKeeper. He needed something fast, distributed, and capable of handling a massive open-source project. GitHub later made Git accessible to everyone.

## When to Use This

> Useful context for understanding why Git exists and why it's designed the way it is — speed, distribution, and open source were not accidents, they were design goals.
